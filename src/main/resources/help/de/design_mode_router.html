Mode conception

Routeur

Le composant Routeur dispose de deux à huit interfaces réseau. Il permet la circulation des paquets IP entre les différents sous-réseaux. Il dispose également d'une fonction de pare-feu qui offre la possibilité de filtrer les paquets qui passent par lui.

Configuration

Nom

Le nom d'un routeur peut être choisi arbitrairement et il n'a aucune influence sur le fonctionnement de la simulation.

Configurer les interfaces

Ce bouton permet d'afficher la liste des interfaces réseau du routeur.

Utiliser le menu contextuel (clic droit) sur les lignes du tableau pour ajouter, supprimer ou modifier une interface.

La liste déroulante du champ Connecté à permet de choisir le nœud à connecter. Il peut s'agir d'un poste actuellement non connecté ou d'un nœud déjà connecté à un autre port du routeur. Dans ce second cas, une permutation des câbles est réalisée.

Sélectionner une ligne du tableau a pour effet d'afficher le câble correspondant en bleu.

Pare-feu

Ce bouton permet de configurer le pare-feu.

Pare-feu

Il est à noter qu'en mode simulation, le pare-feu peut également être configuré via une interface web.

Table de routage

Ce bouton permet d'afficher la table de routage qui contient l'ensemble des règles de routage.

Table de routage

Passerelle

Ce champ permet de saisir l'adresse IP du routeur auquel seront envoyés tous les paquets pour lesquels aucune autre règle de routage ne s'applique. Lorsque ce champ est rempli, une ligne supplémentaire apparait en haut de la table de routage. Il s'agit de la route par défaut.

Le routeur passerelle doit obligatoirement se trouver dans l'un des réseaux auxquels le routeur courant a directement accès.

Routage automatique

En cochant cette case, le routeur gère entièrement le routage des paquets IP. La passerelle et la table de routage manuelle sont ignorées.

Le routage automatique est réalisé à l'aide du protocole RIP qui permet au routeur de communiquer avec les autres routeurs du réseau.
