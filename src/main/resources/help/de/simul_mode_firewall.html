Mode simulation

Pare-feu

L'application pare-feu d'un poste permet de protéger ce poste en rejetant certains paquets IP entrants.

Cette application ne doit pes être confondue avec le pare-feu des routeurs qui opère sur les paquets en transit. Pour plus d'information sur ce type de pare-feu, on pourra consulter la page consacrée aux routeurs.

Lorsque le pare-feu est désactivé, aucun filtrage n'est réalisé et tous les paquets IP entrants sont donc acceptés.

Lorsque le pare-feu est activé, les paquets entrants ne sont acceptés que s'ils satisfont à l'une des règles de filtrage définies. En l'absence de règle, tous les paquets seront donc rejetés.

Règles de filtrage

Le pare-feu accepte ou rejette les paquets IP entrants en se basant sur des règles définies par l'utilisateur.

Chaque règle précise un protocole, un numéro de port et un domaine source.

Le protocole peut être TCP, UDP ou les deux (*)

Le numéro de port peut être n'importe quel nombre compris entre 0 et 65535.

Le domaine source peut être la totalité du réseau ou uniquement le sous-réseau auquel le poste appartient.

L'ordre des règles n'est pas important.

Lorsque le pare-feu est actif, tout paquet IP entrant est intercepté. Le pare-feu parcourt alors la liste des règles de filtrage. Dès que l'une d'elles est satisfaite, le paquet est transmis à l'application destinataire. Si par contre aucune règle n'est satisfaite, le paquet est rejeté.

Rejeter les paquets ICMP (Ping, etc.)

Lorsque cette case est cochée, les paquets entrants de type ICMP seront rejetés. En particulier, le poste ne répondra pas aux Pings.

Journalisation

Le journal permet de suivre les activités du pare-feu.

État du pare-feu

Cocher la case Activer le pare-feu pour rendre ce dernier opérationnel.

Un poste dont le pare-feu est activé est aisément identifiable grâce à son icône.
