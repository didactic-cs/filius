Design mode

DHCP service

The DHCP service assigns any station that requests it an IP address and a subnet mask. It can optionally also provide the IP address of a gateway and the IP address of a DNS server.

Setup

Start IP address / End IP address

These two IP addresses delimit the range of IP addresses which will be assigned to the stations which will request them. These addresses must be in the same subnet as the server.

For network components whose IP addresses are not assigned automatically by a DHCP server, it should be ensured that the addresses used are not within the range of addresses distributed by the server.

Subnet mask

This read-only field recalls the subnet mask of the server. It is also this mask which is communicated to the clients at the same time as the requested IP address.

Reserved addresses

This button provides access to the list of reserved IP addresses. These are MAC address / IP address pairs allowing a specific IP address to be assigned to certain components which need to always have the same IP address.

In the table, use the context menu (right click) on the rows to add, delete or modify a pair of reserved addresses.

Reserved IP addresses must not lie within the range of automatically distributed IP addresses.

Gateway and DNS server

These two fields are used to define the IP address of a gateway as well as the IP address of a DNS server which will be communicated to client stations.

Identical to the server's parameters

When this box is checked, the server uses its own gateway and DNS server settings rather than those defined above.

Activate the DHCP service

By checking this box, the DHCP service is made active on this station.

Each time the simulation mode is activated, the stations configured as DHCP clients will request a DHCP server to obtain an IP address, a subnet mask, the IP address of a gateway and the IP address of a DNS server.
