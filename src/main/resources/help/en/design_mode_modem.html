Design mode

Modem

Modem components work in pairs. They make it possible to establish a link between two disjoint networks. The two connected networks are then part of the same sub-network; the same IP address cannot therefore be used simultaneously in both.

The two networks connected by the modems can be found in the same FILIUS project, but also in separate projects executed on separate (real) computers.

To establish the connection between two modems, one must be configured as a server and the other as a client. This choice has no impact on the exchanges between them once the connection is established.

Settings

Name

The name of a modem can be chosen arbitrarily and it has no influence on the operation of the simulation.

Port

This field allows you to specify a port number that is used when this modem is in server mode.

Wait for incoming connection request

By checking this box, the modem switches to server mode and waits for a connection from a client modem. An orange dot on its icon indicates that it is waiting for connection.

A server modem can only be connected to one client modem and vice versa.

A modem cannot be both a client and a server.

IP adresse of the remote modem

This field allows you to specify the (real) IP address of the computer on which the (virtual) server modem is running in FILIUS. If the server modem is in the same project as the current modem, you can enter localhost as the IP address.

Port of the remote modem

This field allows you to specify the port number of the server modem to contact.

Connect/Disconnect

This button is used to establish the connection with the server modem. Once the connection is established, the icons of the two modems show a green dot.

This button is also used to terminate a connection, both from the client and from the server.
