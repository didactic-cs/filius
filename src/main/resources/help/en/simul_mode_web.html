Mode simulation

Navigateur et serveur web

Serveur web

Ce serveur reproduit le fonctionnement d'un service HTTP.

Général

Le Nom d'hôte reprend en général le nom pleinement qualifié du poste sur lequel le serveur est installé.

Le Répertoire racine est le chemin absolu vers le dossier contenant les pages web du serveur.

Il est possible d'utiliser des hôtes virtuels qui apparaîtront comme d'autres services web. Pour cela, créer un sous-dossier dans le répertoire /webserver puis, dans la boîte de dialogue Hôtes virtuels, ajouter une ligne associant ce répertoire à un nom d'hôte.

Pour que les hôtes virtuels fonctionnent, il est nécessaire qu'un serveur DNS ait été configuré et activé.

Le Fichier index spécifie le nom du fichier qui doit être utilisé lorsqu'une requête porte sur un répertoire et non sur une page web.

La case à cocher En l'absence d'index, lister le répertoire spécifie le comportement du serveur lorsqu'il reçoit une requête vers un répertoire qui ne contient pas le fichier index. Si la case est décochée, le serveur renvoie un message d'erreur du type page non trouvée. Si la case est cochée, le serveur renvoie une page qui liste le contenu du répertoire.

Journalisation

Le journal permet de suivre les activités du service web.

État du service web

Cocher la case Activer le service web pour rendre ce dernier opérationnel.

Un poste dont le service web est activé est aisément identifiable grâce à son icône.

Navigateur web

Cette application affiche les pages web et permet de naviguer entre elles en suivant les liens hypertextes qu'elles contiennent.

Le cache du navigateur est recréé à chaque chargement de page. Ce cache sert uniquement à éviter de télécharger plusieurs fois les images qui apparaissent plusieurs fois dans la page.
