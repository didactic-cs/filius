Mode conception

Modem

Les composants Modem fonctionnent par deux. Ils permettent d'établir un lien entre deux réseaux disjoints. Les deux réseaux raccordés font alors partie d'un même sous-réseau ; une même adresse IP ne pourra donc pas être utilisée simultanément dans les deux.

Les deux réseaux raccordés par les modems peuvent se trouver dans le même projet FILIUS, mais également dans des projets distincts exécutés sur des postes informatiques (réels) distincts.

Pour établir la connexion entre deux modems, l'un doit être configuré comme serveur et l'autre comme client. Ce choix n'a pas de répercussion sur les échanges une fois la connexion établie.

Configuration

Nom

Le nom d'un modem peut être choisi arbitrairement et il n'a aucune influence sur le fonctionnement de la simulation.

Port

Ce champ permet de spécifier un numéro de port qui est utilisé lorsque ce modem est en mode serveur.

Attendre les connections entrantes

En cochant cette case, le modem passe en mode serveur et attend la connexion d'un modem client. Une pastille orange sur son icône indique qu'il est en attente de connexion.

Un modem serveur ne peut être connecté qu'à un seul modem client et inversement.

Un modem ne peut pas être à la fois client et serveur.

Adresse IP du modem distant

Ce champ permet de préciser l'adresse IP (réelle) du poste sur lequel le modem serveur (virtuel) s'exécute dans FILIUS. Si le modem serveur est dans le même projet que le modem courant, on pourra saisir localhost en guise d'adresse IP.

Port du modem distant

Ce champ permet de spécifier le numéro du port du modem serveur à contacter.

Connecter/Déconnecter

Ce bouton permet d'établir la connexion avec le modem serveur. Une fois la connexion établie, les icônes des deux modems présentent une pastille verte

Ce bouton permet également d'interrompre une connexion, aussi bien depuis le client que depuis le serveur.
