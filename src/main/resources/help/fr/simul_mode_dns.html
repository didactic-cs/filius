Mode simulation

Serveur DNS

Le service de noms de domaines (DNS : Domain Name Service) a pour fonction de résoudre les nom de domaines en adresses IP. Le serveur reçoit des requêtes comportant un nom de d'hôte et renvoie une adresse IP.

Résolution directe des noms d'hôtes

Le serveur DNS peut résoudre lui-même certains noms d'hôtes. Il dispose pour cela d'une table d'enregistrements de type A.

Chaque enregistrement de type A contient un nom de d'hôte pleinement qualifié accompagné de l'adresse IP associée.

Délégation de la résolution à un serveur de noms

Un serveur DNS qui ne peut pas résoudre lui-même un nom d'hôte peut solliciter d'autres serveurs. Il doit pour cela disposer d'une table d'enregistrements de type NS.

Chaque enregistrement de type NS contient un nom de domaine accompagné du nom du serveur qui en a la charge. Ce nom de serveur doit être présent dans la table des enregistrements de type A.

Le serveur sollicité peut ne pas être en mesure de résoudre le nom. Dans ce cas, si l'option de résolution récursive est activée, ce serveur pourra solliciter lui-même un autre serveur de noms. La délégation pourra éventuellement se produire plusieurs fois avant que le nom puisse être finalement résolu.

Domaines de messagerie

Pour la transmissions de certain courriels, les serveurs SMTP ont besoin de connaitre l'adresse IP du serveur SMTP du destinataire. Ils sollicitent pour cela le serveur DNS qui dispose à cette fin d'une table d'enregistrements de type MX.

Chaque enregistrement de type MX contient un nom de domaine pleinement qualifié accompagné du nom du serveur SMTP associé.
