Mode conception

Table de routage

La table de routage est constituée d'une liste de règles, appelées routes, qui sont appliquées aux paquets qui parviennent au routeur. Ces règles permettent de déterminer les routes à suivre pour acheminer les paquets.

Configuration

Utiliser le menu contextuel sur les lignes de la table pour ajouter, supprimer ou modifier une règle.

L'ordre dans lequel les règles sont saisies n'est pas important. Le routeur détermine en effet pour chaque paquet qui lui parvient la route la plus spécifique qui s'applique.

IP de destination / Masque

Utilisés conjointement, ces deux paramètres déterminent un sous-réseau ou un hôte précis (dans le cas où le masque vaut 255.255.255.255) que la règle permet d'atteindre.

Passerelle suivante

Ce champ doit contenir l'adresse IP du routeur auquel seront transmis les paquets dont l'adresse de destination est dans le réseau défini par l'IP de destination et le masque spécifiés dans la règle.

La passerelle suivante doit nécessairement se trouver dans un sous-réseau adjacent au routeur, c'est-à-dire auquel il est directement raccordé.

Interface de sortie

Ce champ contient l'adresse IP de l'interface du routeur qui est raccordée au sous-réseau contenant la passerelle suivante.

Règle par défaut

Lorsque l'adresse IP et le masque sont égaux à 0.0.0.0, la règle correspond à la route par défaut. Il s'agit de la règle la moins spécifique et donc celle qui est utilisée en dernier ressort, lorsqu'aucune autre règle ne peut être appliquée.

La règle par défaut peut être générée automatiquement par FILIUS. Il suffit pour cela de remplir le champ Passerelle situé sous le bouton Table de routage du panneau de configuration du routeur.

Règles présaisies

Trois types de règles sont présaisies par FILIUS :

  • La règle localhost 127.0.0.0/255.0.0.0 qui correspond au routeur lui-même.
  • Pour chaque réseau adjacent, la règle d'accès à l'interface correspondante du routeur.
  • Pour chaque réseau adjacent, la règle de sortie vers ce réseau.

Ces règles sont suffisantes pour permettre à un réseau n'utilisant qu'un seul routeur de fonctionner.
